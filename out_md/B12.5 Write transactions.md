Chapter B12. Memory
B12.5. Write

• For WriteBackPtl with TagOp :
– Transfer: Not permitted.


• For WriteNoSnpFull with TagOp :


– Update: Not permitted.
– Match: Not permitted.


– Transfer: TU bits are inapplicable and must be zero. Clean tag transfer is permitted from the Request
Node to the Home Node as well as the Home Node to the Subordinate Node.


• For WriteNoSnpDef with TagOp :


– Update: All TU bits must be 1.
– Match: TU bits are inapplicable and must be zero.


– Transfer: Not permitted.
– Update: Not permitted.


• For WriteUniqueFull and WriteUniqueFullStash with TagOp :
– Match: Not permitted.


– Transfer: Not permitted.
– Update: All TU bits must be 1.


• For WriteNoSnpPtl , WriteUniquePtl , and WriteUniquePtlStash with TagOp :
– Match: TU bits are inapplicable and must be zero.


– Transfer: Not permitted.
– Update: Any combination of TU and BE bits can be 1, including all ones or all zeros.


• For WriteEvictFull and WriteEvictOrEvict with TagOp :


– Match: TU bits are inapplicable and must be zero. Tag Match must be performed for only those tags
that have at least one corresponding BE = 1. A Tag Match must not be performed when all BE bits are
set to 0.


– Transfer: Clean tags must be returned. TU bits are inapplicable and must be zero.
– Update: Not permitted.


• For WriteNoSnpZero and WriteUniqueZero only TagOp
– Match: Not permitted.
Invalid is permitted.
For a Write request with a TagOpMatch of the tags within the size wrap boundary can take any value, and the tags
outside of size are inapplicable and can also take any value.
In the WriteDataCancel Write data response, irrespective of the TagOp value in the Write request, the MTE fields
are inapplicable and must be zero.
In Write data, with TagOp Invalid, all the TU bits and all the Tag bits must be zero.
